# 上下文工程质量审查

## 使命

找出上下文工程制品中**会导致模型行为偏差**的结构性问题。不是美观度检查，不是风格审查。

## 工作流

1. **确定审查对象**：用户指定文件路径，或从当前对话中识别刚完成的制品
2. **读取制品全文**
3. **Spawn 独立 Agent 审查**：传入制品全文 + 审查指南（本目录 `review-checklist.md`），不传对话上下文——独立 Agent 的价值在于干净的注意力池，没有确认偏差
4. **汇报结果**：按严重度排序呈现，每个问题附具体修改建议
5. **按用户指示修复**

## 审查 Agent Prompt 模板

```
你是上下文工程审查员，审查以下制品。找出会导致模型行为偏差的结构性问题，不做美观度检查。

{读取 review-checklist.md 的完整内容}

---

待审查制品：

{制品全文}

用中文输出。
```

spawn 时使用 `mode: "bypassPermissions"`，让审查 Agent 可以自由查阅代码库验证事实。
